feat: Phase 2.5 webhook routing, context persistence, and UI enhancements

This comprehensive update implements dual-write webhook routing, context
persistence, improved error handling, and dashboard enhancements for Phase 2.5.

## Core Features

### 1. Dual-Write Webhook Routing
- All Phase 2 webhook endpoints now forward to Phase 2.5 orchestrator
- Non-blocking: Phase 2 succeeds even if Phase 2.5 fails
- Enables Phase 2.5 to receive all webhook traffic without TradingView config changes
- Files: src/app/api/webhooks/{signals,saty-phase,trend}/route.ts

### 2. Flexible Webhook Adapters (Priorities 1-3)
- Signal adapter: Accepts minimal format (ticker, trend, score)
- SATY adapter: Ultra-flexible parsing from minimal data (symbol, timeframe, bias)
- Endpoint detector: Auto-detects webhook type with confidence scoring
- TradingView text wrapper support: Parses {"text": "{...}"} format
- Files: src/webhooks/{signalAdapter,satyAdapter,endpointDetector}.ts

### 3. Context Persistence & Status API
- Persist latest context snapshot per symbol to database
- New GET /api/phase25/context/status endpoint
- Shows completeness, source freshness, and latest snapshot
- Files: src/phase25/utils/contextDb.ts, src/app/api/phase25/context/status/route.ts

### 4. Context Store Improvements
- Default timeout increased to 30 minutes (env-configurable)
- Auto-persist context snapshots to DB on updates
- Better completeness tracking and expiration handling
- File: src/phase25/services/context-store.service.ts

### 5. Ledger Write Visibility & Retry
- Added details.ledgerStored + details.ledgerError in webhook responses
- 2-attempt retry for ledger append failures with 200ms delay
- Better error tracking and visibility
- File: src/phase25/services/decision-orchestrator.service.ts

### 6. Dashboard Enhancements
- New Phase25ContextStatus component showing completeness/freshness
- Engine version filter in /api/decisions endpoint
- Display engine version in Phase25DecisionCard and Phase25HistoryTable
- Files: src/components/dashboard/Phase25ContextStatus.tsx, src/app/page.tsx

### 7. Security & Best Practices
- Replaced Math.random() with crypto.randomUUID() for secure UUID generation
- File: src/ledger/ledger.ts

### 8. Validation Endpoint (Priority 6)
- New POST /api/webhooks/validate for testing payloads
- Auto-detects webhook type and validates structure
- Returns detailed error messages with examples
- File: src/app/api/webhooks/validate/route.ts
